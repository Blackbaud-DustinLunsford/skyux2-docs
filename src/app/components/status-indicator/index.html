The status indicator classes create styled text to draw attention to data that needs to be addressed. A status is "a state or condition at a particular time". Statuses fall into four categories: success, warning, danger, and info. Examples include late, submitted, awaiting approval, etc.

UX guidelines
Status importance

Some statuses are crucial to a user's task, while others are no more important than other information on the page or page area. If they are crucial to a user's task you should use a status indicator. If they are ancillary, treat them as regular text.

Just because one status in an area is crucial, does not mean they all are. For example, a list of invoices could show a mix of paid, unpaid and overdue invoices. It's likely that in that case only the overdue invoices would need a status indicator if they are the only invoices that need immediate attention and action.

Status indicator placement

Status indicators should be used in an inline context tied to a specific piece of data, rather than in a summary area. For example, an indicator can be placed in a grid cell or card. If you need to convey a higher-level status in a summary area, use a label instead.
